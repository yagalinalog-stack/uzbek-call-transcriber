# Uzbek Call Transcriber

Сервис для транскрибации аудио и видео файлов с узбекской речью в структурированный текст с переводом на русский язык.

## Что делает сервис

- принимает файлы `mp3`, `wav`, `m4a`, `ogg`, `mp4`;
- загружает файл через Express API;
- отправляет аудио/видео в Gemini Files API;
- получает структурированный JSON с:
  - количеством спикеров;
  - кратким резюме звонка;
  - ключевыми моментами;
  - диалогом по репликам;
  - оригинальным узбекским текстом;
  - смысловым переводом на русский язык;
- позволяет просматривать результат в интерфейсе;
- экспортирует транскрипт в `TXT` и `DOCX`.

## Текущая версия

`v1.0` - первая рабочая версия сервиса.

Ключевые улучшения первой версии:

- загрузка больших файлов через временное disk storage вместо хранения всего файла в памяти;
- лимит загрузки настраивается через `MAX_UPLOAD_MB`;
- корректная нормализация MIME-типа для Gemini Files API;
- увеличенный лимит ответа модели для длинных транскрипций;
- JSON-ошибки для некорректных форматов, превышения размера и сбоев обработки;
- автоматическая очистка временных локальных файлов и файлов Gemini Files API.

## Архитектура

```text
React UI -> Express API -> temporary upload file -> Gemini Files API -> Gemini model -> JSON result -> UI/export
```

Основные файлы:

- `server.ts` - Express-сервер, загрузка файлов, интеграция с Gemini, обработка ошибок;
- `src/App.tsx` - пользовательский интерфейс загрузки и просмотра транскрипта;
- `src/docxExport.ts` - экспорт результата в Word;
- `src/types.ts` - типы результата обработки.

## Требования

- Node.js 22+
- Gemini API key

## Переменные окружения

Создайте `.env` на основе `.env.example`:

```env
GEMINI_API_KEY="YOUR_GEMINI_API_KEY"
MAX_UPLOAD_MB=200
GEMINI_MODEL="gemini-3.5-flash"
GEMINI_MAX_OUTPUT_TOKENS=32768
GEMINI_FILE_PROCESSING_ATTEMPTS=150
```

Описание:

- `GEMINI_API_KEY` - ключ Gemini API;
- `MAX_UPLOAD_MB` - максимальный размер загружаемого файла;
- `GEMINI_MODEL` - модель Gemini для анализа аудио;
- `GEMINI_MAX_OUTPUT_TOKENS` - лимит ответа модели;
- `GEMINI_FILE_PROCESSING_ATTEMPTS` - количество проверок готовности файла в Gemini Files API.

## Развертывание

Для полноценного тестирования лучше использовать обычный Node.js web service или VPS:

- Railway;
- Render;
- Fly.io;
- VPS с Node.js, PM2 и nginx.

Vercel для текущей версии не рекомендуется, потому что сервис принимает файлы и может выполнять долгие запросы к Gemini. Serverless-лимиты Vercel по размеру тела запроса и длительности выполнения плохо подходят для транскрибации больших аудио.

## Известные ограничения

Текущая версия обрабатывает один файл одним запросом к Gemini. Для очень длинных разговоров модель может вернуть слишком большой JSON или не уложиться в лимит ответа.

Следующий архитектурный шаг:

- добавить нарезку аудио на чанки через `ffmpeg`;
- транскрибировать чанки отдельно;
- собирать итоговый диалог и резюме отдельным финальным запросом.

## Безопасность

- не коммитьте `.env`;
- не публикуйте `GEMINI_API_KEY`;
- временные файлы удаляются после обработки;
- файл в Gemini Files API удаляется после завершения запроса.
